Part One

1. What is HTTP?

HTTP stands for Hypertext Transfer Protocol. Protocols are the conventions by which one thing talks to another. HTTP is the standard protocol for communication between browsers and servers.

2. What is a URL?

URL stands for Uniform Resource Locator. A URL is the address that specifies the location for any resource on a network.

3. What is a DNS?

DNS stands for Domain Name System. When a client makes a request for a web address/domain name, the DNS looks up the IP address for a domain name and delivers the IP address to the client that made the request.

4. What is a query string?

The query string is extra information added to the url that assigns values to specified parameters. Examples of such info could be search terms, info from forms, a specified database entry, etc.

5. What are two HTTP verbs and how are they different?

Two common HTTP verbs are GET and POST. A GET request will specifies resources which the client would like served back to them. A POST request would is used to submit an entity or information to a specified resource. A GET request has no side effects and POST request has side effects

6. What is an HTTP request?

HTTP requests are methods to indicate a desired action to be performed on a specified resource.

7. What is an HTTP response

An HTTP response is the response from a server upon that server receiving an HTTP request.

8. What is an HTTP header? Give a couple examples of request and response headers you have seen.

The HTTP header is a field in an HTTP request or response that includes additional content and information. Some examples of content included in headers are hostname, date, language, cookies.

9. What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser?

Once the URL is submitted in the address bar of the browser, it will initiate the HTTP, Hypertext Transfer Protocol. The domain name will be sent to the DNS, where the DNS will search for the IP address. The DNS will send the IP address back to the client so the client can then make the GET request to the website's server. The server at that IP address (somesite.com) will receive the GET request for /page.html in the /some/ directory of somesite.com. Assuming the page/resource exits at that URL, the server will server that resource to the client and the user at the client will be able to view “http://somesite.com/some/page.html".
